# ADC Bringup Toolkit — File Summary

## 📦 Package Contents

This toolkit provides a complete workflow for commissioning the 9V supply voltage monitoring system (Requirement 13) on the Shelly ESP32 Logger.

### 1. **ADC_Bringup.ino** (Main Firmware)
**Type:** Arduino sketch for ESP32  
**Size:** ~5 KB compiled  
**Purpose:** Minimal bring-up firmware for characterizing the ADC and voltage divider

**Features:**
- Continuous ADC sampling of GPIO 35 (PIN_VSUPPLY)
- Configurable voltage divider ratio (VDIV_RATIO)
- WiFi enabled for realistic current draw simulation
- Web endpoint `/adc` for remote monitoring
- Serial output with ADC counts, calculated voltage, and status
- Mains loss threshold detection

**OTA Compatible:** Yes ✓  
Can be flashed via `http://braun_PZEM.local/update` to replace main logger

**Key Parameters (Config):**
```cpp
#define VDIV_RATIO               3.0f       // Adjust after initial measurement
#define SHUTDOWN_THRESHOLD_V     5.5f       // Brownout threshold
#define ADC_SAMPLES              16         // Noise reduction via oversampling
```

### 3. **adc_analysis.py** (Python Analysis Tool)
**Type:** Python 3 utility script  
**Dependencies:** matplotlib (optional, for plots)  
**Purpose:** Parse serial logs and calculate optimal VDIV_RATIO automatically

**Features:**
- Parse serial output from ADC_Bringup.ino
- Extract ADC counts and calculated voltages
- Compute statistics (min, max, mean, std dev)
- Automatic VDIV_RATIO calculation from multimeter reading
- Generate calibration plots (PNG)
- Recalculate voltages with different ratios

**Usage:**
```bash
# Basic analysis
python3 adc_analysis.py --serial-log output.txt

# With multimeter calibration
python3 adc_analysis.py --serial-log output.txt --multimeter 8.95

# Generate plot
python3 adc_analysis.py --serial-log output.txt --multimeter 8.95 --plot
```

**For:** Data analysis and validation of measurements

---

### 4. **Logger_ADC_Integration.h** (Integration Template)
**Type:** C++ code template / documentation  
**Size:** ~200 lines (well-commented)  
**Purpose:** Template showing how to integrate calibrated ADC into main Logger.h

**Contents:**
- `adcToVoltage()` – ADC count to voltage conversion function
- `checkMainsVoltage()` – Mains loss detection & flush logic
- Member variables needed in Logger class
- Constructor initialization additions
- Integration points in main loop()
- Testing checklist

**Usage:**
1. Run ADC_Bringup.ino and calibrate (get VDIV_RATIO)
2. Copy functions from this template into Logger.h
3. Add constants to Config.h
4. Call `logger.checkMainsVoltage()` from main loop()
5. Re-flash main firmware

**For:** Firmware developers integrating ADC monitoring into production code

```
             │
             ▼

│  3. python3 adc_analysis.py ... --multimeter 8.95  │
│  4. Review statistics and generated plots          │
└────────────┬────────────────────────────────────────┘
             │
             ▼
┌─────────────────────────────────────────────────────┐
│ PHASE 4: Integration (use Logger_ADC_Integration)  │

│  5. Compile and flash main firmware via OTA        │
│  6. Test: voltage drop triggers flush & LED blink  │
└─────────────────────────────────────────────────────┘
```

## 🛠️ Getting Started

### For Commissioning Engineers:

1. **Start here:** README_ADC_Bringup.md (5 min read)
2. **Follow:** ADC_Calibration_Guide.md (step-by-step)
3. **Flash:** ADC_Bringup.ino via Arduino IDE or OTA
4. **Measure:** Multimeter + serial monitor
5. **Analyze:** `adc_analysis.py --serial-log output.txt --multimeter 8.95`

### For Firmware Developers:

1. **Review:** Logger_ADC_Integration.h
2. **Extract:** adcToVoltage() and checkMainsVoltage()
3. **Integrate:** Add to Logger.h and main loop()
4. **Update:** Config.h with calibrated VDIV_RATIO
5. **Test:** Run main firmware + validate brownout detection

### For Project Managers:

- **Effort:** 1–2 hours per hardware variant
- **Cost:** Zero (existing ESP32 + resistor divider)
- **Risk:** Low (diagnostic firmware, easy rollback)
- **Requirement Met:** Req 13 (Supply Monitoring)
